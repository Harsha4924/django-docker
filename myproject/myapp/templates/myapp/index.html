Docker

Docker uses the concept of containerization, we package our application with dependencies. so that application can run in any environments without any conflicts!, and the containers are light weight because they use the resources from host os and they dont hold the os just like V.M. On an EC2, if we are not using the docker, the application dependencies will directly install on ec2 which might cause dependency issues with in the applications, and if we are using the docker, each application has the container and each container has its own dependencies and there will be no conflicts. for production env, its better to use the docker

This is the breif of Docker
